3. Out of Scope for v1

The following are explicitly excluded from the MVP Build Fee. Any of these can be added later as a change order or v1.1 engagement.

  • Native mobile apps (iOS / Android). The MVP is a responsive web app.
  • AI features (recommendations, matchmaking, semantic search) beyond simple keyword search.
  • Custom branding / design system beyond a clean, accessible Ecodia-default look-and-feel. Major brand design work is a separate engagement.
  • Integrations with third-party systems (other than Stripe and the chosen email/auth provider).
  • Multi-language / internationalisation.
  • Data migration from any existing CETN systems.
  • Content creation (marketing copy, images, video, member onboarding content) — provided by the Client.
  • Ongoing marketing, SEO, and growth work.

4. Technical Stack

The MVP will be built on Ecodia's standard stack:

  • Frontend: Next.js 14+ (App Router), Tailwind CSS, TypeScript.
  • Backend: Next.js API routes (or lightweight Node service if needed).
  • Database and auth: Supabase (PostgreSQL + Auth + Storage).
  • Hosting: Vercel (frontend), Supabase (database and auth).
  • Payments (if F10 included): Stripe (AUD).
  • Email: Resend or Postmark (transactional).
  • Analytics: Plausible or PostHog (lightweight, privacy-respecting).
  • Source control: GitHub repository under the EcodiaTate organisation, private.

6. Acceptance Criteria

At M5, the MVP is accepted if, in Client's reasonable assessment:

  1. All features marked "Included" in section 2 are present and materially functioning.
  2. The MVP is deployed to a production URL accessible to the Client's admin account.
  3. Basic responsive layout works on current versions of Chrome and Safari, desktop and mobile.
  4. No known critical bugs that prevent a user from signing up, signing in, and using core directory and project features.

7. Assumptions and Dependencies

  • Client provides all content (text, images, logos, copy) in usable form at or before the relevant milestone.
  • Client responds to Ecodia feedback requests within 5 business days during active build phases.
  • Client owns (or has permission to use) all content and branding supplied.
  • A domain name is either supplied by the Client or purchased by Ecodia and on-billed as a small reimbursable cost.
  • Third-party services (Vercel, Supabase, Stripe, Resend/Postmark, analytics) are available and on their standard pricing as of the signing date.
