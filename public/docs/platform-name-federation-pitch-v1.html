02  /  The pattern

A peak-body federation runs on spreadsheets, email, and WhatsApp groups across 5 to 300 sub-orgs.

State-level Landcare networks. Conservation Volunteers Australia. Regional NRM bodies. Junior Landcare's 12,000+ schools and groups on the National Landcare Directory. Coastcare, Bushcare, Rivercare. Trust-for-Nature covenantor cohorts.

Same shape every time. People show up. They organise into local groups. They run events on the ground. They record what happened. The result has to land somewhere a funder, a board, and a regulator can read.

The infrastructure layer underneath this is missing. The work is held together by individual volunteers carrying the seams between four or five separate tools. When a coordinator changes role, the institutional memory walks out with the laptop.

The 6,000 Landcare groups, 100,000 to 140,000 volunteers, and 12,000 Junior Landcare schools quoted in Landcare Australia's 2023/2024 Annual Report describe a federation problem with no current platform answer.

04  /  What we ship

One codebase. Multi-tenant. Native iOS and Android. Reverse-sync to your existing spreadsheets.

  • Multi-tenant platform. Each peak body gets its own deployment with its own brand, its own configurable impact metrics, its own legal pack, its own role hierarchy. Same generic core, no per-tenant rewrite.
  • Native iOS and Android from one codebase. Capacitor wrapping a React frontend. Offline-capable on phones because most events happen in places without signal. Same code ships the web admin.
  • Excel and SharePoint reverse-sync. Bidirectional, idempotent, runs on a Supabase Edge Function on a 30-minute cron. Preserves Microsoft Forms integer-ID rows untouched while writing UUID app rows back to the same sheet. Conflict surfacing for admin reconciliation. In production for over six months at our anchor tenant.
  • Per-tenant brand reskin in 8 to 12 working days. Audited from the live codebase: 83% of frontend code never says the deployment's name. The 17% that does is concentrated in copy, identity strings, and a small set of org-specific pages. The structural shape is brand-neutral.
  • Federation roll-up dashboard. Members belong to local branches. Branches roll up to regions. Regions roll up to a national view. Impact metrics aggregate at every level.
